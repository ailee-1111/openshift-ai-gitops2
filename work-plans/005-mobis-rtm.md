# Mobis RHOAI PoC 요구사항 추적 매트릭스 (RTM)

- 작성일: 2026-05-15
- 최종 수정: 2026-05-15
- 고객: Mobis (현대모비스)
- PoC 기간: 2026-05 2주차 ~ 06 1주차 (약 4주)
- 인프라: 데이터사이언스팀 HGX(H200) 1대
- 모델: SmolLM2-135M (PoC 검증용 경량 모델)

## Why (왜 이 문서가 필요한가)

고객 요구사항(No.1~85)을 시나리오(S1~S6)로 그룹핑하고, 각 항목의 구축 런북·검증 런북·IaC를 매핑한다. RTM은 gap analysis 도구를 겸하며, 런북이 없는 항목(❌)은 신규 작성이 필요하다.

## Tradeoffs (각 옵션의 장단점)

Core 34개를 우선 실행하고, Advanced 21개를 순차 진행한다. Exploratory 24개는 시간 허용 시 수행.

## Decision (무엇을 선택했고 그 이유)

Core + Advanced를 S1~S6 시나리오 순서대로 실행한다. 검증 실행 순서:

```
Phase 0: 사전 구성 (runbooks/40) — 0.5일
Phase 1-1: S1 모델 관리 (runbooks/60, 70) — 2~3시간
Phase 1-2: S2 Pipeline (runbooks/61, 71) — 3~4시간
Phase 1-3: S3 Auto-scaling (runbooks/62, 72) — 2~3시간
Phase 1-4: S4 장애 복구 (runbooks/63, 73) — 1~2시간
Phase 1-5: S5 Scale-to-Zero (runbooks/64, 74) — 2~3시간
Phase 1-6: S6 운영관리 (runbooks/65, 75) — 3~4시간
Phase 2: 종합 검증 (runbooks/80) — 0.5일
Phase 3: 리포팅 (reports/mobis/) — 0.5일
```
