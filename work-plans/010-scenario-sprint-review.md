# PoC 시나리오 10회 스프린트 계획서 — 컨설턴트/SA 듀얼 페르소나 리뷰

- 작성일: 2026-05-21
- 원본: `Customer/RHOAI PoC 계획 및 일정.md` (고객 요구사항 85개, 6개 시나리오)
- 검증 데이터: `[현대모비스] RHOAI PoC 시나리오-v2` Google Sheet (85항목 검증 결과)
- 기존 RTM: `work-plans/005-customer-rtm.md`
- 기존 로드맵: `work-plans/009-roadmap-v4.md`

## 목적

고객 원본 시나리오(S1~S6)와 추가 시나리오(S7~S10)의 적절성을 **컨설턴트**(비즈니스 관점)와 **솔루션 아키텍트**(기술 관점) 두 페르소나가 10회 스프린트에 걸쳐 검토한다. 매 스프린트 종료 시 각 페르소나가 5개 보완 내역을 제시하고, 다음 스프린트는 해당 보완을 반영하여 시나리오를 현실적·실현 가능·검증 정확한 수준으로 고도화한다.

## 페르소나 정의

| 역할 | 관점 | 평가 기준 |

| **컨설턴트 (C)** | 비즈니스 가치, 고객 요구 충족, ROI, 리스크 관리, 일정 현실성 | 고객이 받는 가치가 명확한가? 일정 내 완료 가능한가? |

```
                                                        │

```

### S7 MaaS 컴포넌트 장애 영향도 (SA-7,SA-18 해소)

### S4 안전 검증 절차 (SA-14,SA-17 해소)

단일 Master 환경 제약 반영:

1. **Pod 삭제 테스트** (안전): Worker 노드의 서빙 Pod만 삭제 → 복구 시간 측정
2. **노드 drain 테스트** (주의): Worker 노드만 drain. Master 절대 건드리지 않음
3. **API 서버 모니터링**: drain 전 `oc get nodes` 응답 확인, drain 중 10초 간격 health check
4. **즉시 복구**: `oc adm uncordon` 준비. 30초 내 API 무응답 시 즉시 uncordon
